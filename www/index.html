// the moat

Your memory, not their server.

Every assistant keeps a memory of how you work. The real question is where it lives — and whether you can read, move, and version it.

auto-learned memorywhere it livesyours to move?
Claude Code + engramyour disk · plain Markdownyes — it's just files in git
GeminiGoogle account · serverno — vendor-locked
GrokxAI account · serverno — vendor-locked
CopilotGitHub account · serverno — vendor-locked

Claude Code is the outlier — it writes memory to ~/.claude as Markdown you own. engram adds the fast UI and the git layer, so your team's context stays portable and never trapped behind a login.

// team sharing

Promote a memory. Your team pulls it.

One git repo your team reads and writes. Sync is manual — engram badges what changed and you decide what's personal and what's team.

// command palette

Two keys run everything: / and @

ctrl+p opens the palette. Type / to jump around engram, or @ to hand the work to Claude.

ctrl+p type / for commands · @ for the assistant

/   jump around engram

/memorybrowse your memories
/plansbrowse plan-mode plans
/filesCLAUDE.md + MEMORY.md · read-only
/settingstheme, editor

@   hand off to Claude

@Claudefix & edit memories with Claude Code

Launches an interactive Claude Code session in the right folder — there you run Claude's own /memory, /memory-check, and the rest. engram reloads when it exits.
