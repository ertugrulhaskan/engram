// what it does

Every memory, one keystroke away.

◧

Two-pane TUI

A searchable list beside a fully rendered markdown preview. Glamour-styled, fast, keyboard-driven.

/

Filter & search

Hit / to fuzzy-filter across titles, projects, and types. Narrow to what matters in a keystroke.

✎

Edit in $EDITOR

Press e to open the selected memory in your editor, then reload in place. n to create, d to delete.

▦

Group your way

Toggle between by-project and by-type with g. Filter to a single type with t.

⌘

Command palette

ctrl+p jumps anywhere. Switch sources and browse plan-mode plans alongside memories.

◑

Live theming

Classic Dark, Tokyo Night, Gruvbox, Nord, Dracula. Switch with 1-5; config persists under XDG.

// zero setup

It already knows where to look.

engram scans ~/.claude/projects/*/memory/ and parses both shapes Claude Code writes — YAML frontmatter and plain markdown with a MEMORY.md index. Nothing to configure.

  • Discovers memories across every project automatically
  • Parses YAML frontmatter and plain-markdown shapes alike
  • Keeps MEMORY.md in sync on new, edit, and delete
  • Reconciles a drifted index with R
